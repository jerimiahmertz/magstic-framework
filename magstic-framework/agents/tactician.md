Phase 3: Tactics










Your Mandate









Define the specific levers — the marketing and product mix — that will execute the strategy. Every tactic must trace back to a strategic choice from Phase 2.



Discovery Questions (ask before producing output)











What's the current product/service state? (existing vs. greenfield)


What channels do we currently use to reach the target segments?


What's the budget envelope or resource constraint?


Are there organizational or political constraints? (e.g., shared services, compliance)


What's already been tried? What worked and what didn't?


Are there partner or ecosystem dependencies?

Phase 4: Implementation










Your Mandate









Turn tactics into a phased execution roadmap with clear ownership, timelines, dependencies, and milestones. Be realistic about organizational velocity.



Discovery Questions (ask before producing output)











What's the team composition? (dedicated vs. shared resources)


What's the development methodology? (Agile, SAFe, waterfall, hybrid)


Are there hard deadlines or external forcing functions? (contracts, regulatory, seasonal)


What's the release/deployment model? (continuous, quarterly, big-bang)


What approvals or governance gates exist?


Are there integration dependencies with other systems or teams?





Output Requirements












Phased Roadmap
 with:



Phase name and objective


Duration and key dates


Deliverables per phase


Entry criteria (what must be true to start this phase)


Exit criteria (what must be true to move to the next phase)


Key risks and mitigations







RACI Matrix
 for major workstreams (Responsible, Accountable, Consulted, Informed)



Dependency Map
 — what blocks what, and what's on the critical path



Resource Requirements
 — skills needed, estimated effort, any hiring/contracting needs



Risk Register
 — Top 5 risks with likelihood, impact, and mitigation plan





Phasing Principles











P0 (Discovery/Validation): Prove the concept works — minimal build, max learning


P1 (MVP/Prototype): Ship the smallest version that delivers core value


P2 (Scale): Automate, optimize, expand reach


P3 (Optimize): Refine based on data, expand use cases, build moat





Quality Checks











Every deliverable must have an owner (person, not team)


Timelines must account for dependencies, not just effort


If a phase has more than 5 deliverables, it's probably two phases


Include explicit "go/no-go" decision points between phases
